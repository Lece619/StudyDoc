1. 요구사항 
	꽃 테이블과 화분 테이블 2개가 필요하고,
	꽃을 구매할 때 화분도 같이 구매한다.
	꽃은 이름과 색깔 가격이 있고
	화분은 제품 번호, 색깔, 모양, 꽃이름이 있다.

2. 개념적 설계(개념적 모델링)
	회원	주문		꽃	화분
	-------------------------------------------------------
	ID	주문번호		꽃 이름	제품번호
				꽃 색깔
	-------------------------------------------------------
	PW	주문일자		가격	색깔
	이름	아이디			모양
	주소	꽃 색깔			꽃이름
		제품번호

3. 논리적 설계(논리적 모델링)
	회원	주문		꽃		화분
	----------------------------------------------------------------
	ID(PK)	주문번호(PK)	꽃 이름(PK)	제품번호(PK)
				꽃 색깔(PK)
	----------------------------------------------------------------
	PW	주문일자		가격		색깔
	이름	아이디(FK)			모양
	주소	꽃 색깔(FK)			꽃이름(FK)
		제품번호(FK)

4. 물리적 설계(물리적 모델링)
	
	USER
	USER_ID : VARCHAR2(100)
	---------------------------------
	USER_PW : VARCHAR2(100)
	USER_NAME : VARCHAR2(200)
	USER_ADDRESS : VARCHAR2(100)

	ORDER
	ORDER_NUM : NUMBER
	--------------------------------------
	ORDER_DATE : DATE
	USER_ID : VARCHAR2(100)
	FLOWER_COLOR : VARCHAR2(100)
	PRODUCT_NUM : NUMBER
	
	FLOWER 
	FLOWER_NAME : VARCHAR2(100)
	FLOWER_COLOR : VARCHAR2(100)
	--------------------------------------
	FLOWER_PRICE : NUMBER

	PRODUCT
	PRODUCT_NUM : NUMBER
	---------------------------------
	PRODUCT_NAME : VARCHAR2(300)
	PRODUCT_COLOR : VARCHAR2(100)
	PRODUCT_SHAPE : VARCHAR2(100)
	FLOWER_COLOR : VARCHAR2(100)
