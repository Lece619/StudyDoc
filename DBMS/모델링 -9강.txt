모델링
	추상적인 주제를 DB에 맞게 설계하는 것

1. 요구사항 분석
	회원, 주문, 상품 : 3가지를 관리하고자 한다.

2. 개념적 설계( 개념 모델링)
	회원	주문	상품
---------------------------------------------------
	ID	주문번호	상품번호		<=PK
---------------------------------------------------
	PW	주문일자	상품명
	이름	ID	가격
	주소	상품번호	재고량
	이메일
	생일


3. 논리적 설계(논리 모델링)
	회원	주문		상품		결제정보
--------------------------------------------------------------------------------
	ID(PK)	주문번호(PK)	상품번호(PK)	계좌번호(PK)	<=PK
						은행명(PK)   ==조합키
--------------------------------------------------------------------------------
	PW	주문일자		상품명		예금주
	이름	ID(FK)		가격		CVC
	주소	상품번호(FK)	재고량		
	이메일	계좌번호(FK)
	생일

4. 물리적 설계(물리 모델링)

	USER
	USER_ID : VARCHAR2(100)
	---------------------------------
	USER_PW : VARCHAR2(100)
	USER_NAME : VARCHAR2(200)
	USER_EMAIL : VARCHAR2(300)
	USER_BIRTH : DATE

	PRODUCT
	PRODUCT_NUM : NUMBER
	---------------------------------
	PRODUCT_NAME : VARCHAR2(300)
	PRODUCT_PRICE : NUMBER
	PRODUCT_COUNT : NUMBER

	ORDER
	ORDER_NUM : NUMBER
	--------------------------------------
	ORDER_DATE : DATE
	USER_ID : VARCHAR2(100)
	PRODUCT_NUM : NUMBER

1. 요구사항 
	꽃 테이블과 화분 테이블 2개가 필요하고,
	꽃을 구매할 때 화분도 같이 구매한다.
	꽃은 이름과 색깔 가격이 있고
	화분은 제품 번호, 색깔, 모양, 꽃이름이 있다.

2. 개념적 설계(개념적 모델링)
	회원	주문		꽃	화분
	-------------------------------------------------------
	ID	주문번호		꽃 이름	제품번호
				꽃 색깔
	-------------------------------------------------------
	PW	주문일자		가격	색깔
	이름	아이디			모양
	주소	꽃 색깔			꽃이름
		제품번호

3. 논리적 설계(논리적 모델링)
	회원	주문		꽃		화분
	----------------------------------------------------------------
	ID(PK)	주문번호(PK)	꽃 이름(PK)	제품번호(PK)
				꽃 색깔(PK)
	----------------------------------------------------------------
	PW	주문일자		가격		색깔
	이름	아이디(FK)			모양
	주소	꽃 색깔(FK)			꽃이름(FK)
		제품번호(FK)

4. 물리적 설계(물리적 모델링)
	
	USER
	USER_ID : VARCHAR2(100)
	---------------------------------
	USER_PW : VARCHAR2(100)
	USER_NAME : VARCHAR2(200)
	USER_ADDRESS : VARCHAR2(100)

	ORDER
	ORDER_NUM : NUMBER
	--------------------------------------
	ORDER_DATE : DATE
	USER_ID : VARCHAR2(100)
	FLOWER_COLOR : VARCHAR2(100)
	PRODUCT_NUM : NUMBER
	
	FLOWER 
	FLOWER_NAME : VARCHAR2(100)
	FLOWER_COLOR : VARCHAR2(100)
	--------------------------------------
	FLOWER_PRICE : NUMBER

	PRODUCT
	PRODUCT_NUM : NUMBER
	---------------------------------
	PRODUCT_NAME : VARCHAR2(300)
	PRODUCT_COLOR : VARCHAR2(100)
	PRODUCT_SHAPE : VARCHAR2(100)
	FLOWER_COLOR : VARCHAR2(100)
